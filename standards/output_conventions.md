# Output Conventions

## Objectives

Outputs should be easy to review, trace to source material, and compare across projects.

## Requirements

- Include project name and document date where known.
- Separate facts, assumptions, and inferred conclusions.
- Prefer Markdown for narrative summaries and JSON for structured data.
- Preserve units exactly as stated in source material.
- Note missing inputs or unresolved ambiguities explicitly.

## Recommended Sections

1. Document Summary
2. Key Findings
3. Risks or Flags
4. Open Questions
5. Source Traceability
